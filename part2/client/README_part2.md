# Mini-Project – Part 2  
## Custom MCP Server + Multi-Server Orchestration

This folder contains the full implementation for **Part 2** of the MCP Mini-Project.  
The objective of this part is to build a **custom MCP server** that exposes domain-specific tools (Raw Dessert Texture Intelligence), integrate it with **two external MCP servers**, and orchestrate all tools via an **LLM-driven MCP client**.

---

##  Objectives

- Build a custom MCP server using the official MCP Python SDK.
- Expose non-trivial tools (texture analysis, raw dessert advice).
- Connect to at least two third-party MCP servers:
  - **arXiv MCP server** → scientific paper search
  - **Notes/Markdown MCP server** → save results
- Implement an LLM-controlled agent capable of:
  - Tool discovery  
  - Multi-step planning  
  - Context accumulation  
  - Error-handling and retries  
- Produce a final structured output saved as `final_raw.md`.

##  How to Run the Project

### 1. Activate the virtual environment
```bash
source venv/bin/activate   # Mac/Linux
.\venv\Scripts\activate    # Windows

2. Install dependencies
pip install -r requirements.txt

3. Start the custom MCP server (raw texture server)
python servers/raw_server.py

4. Run the client orchestrator
python client/agent_client.py


The client will:

Connect to:

raw_server (custom)

arxiv_server (external)

notes_server (external)

Ask the LLM to plan tool calls

Fetch relevant scientific inputs

Clean and interpret them

Generate specialized texture advice

Save results to final_raw.md

 Example Goal Used

“tiger nut hydration behavior”

Pipeline executed:

 Search scientific papers → arXiv server

 Clean & extract insights → raw_server

 Generate raw dessert texture advice

 Save output → notes_server

 Final file: final_raw.md
